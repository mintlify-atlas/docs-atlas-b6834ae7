---

The Mercado Pago payment endpoints provide the same hotspot-provisioning experience as the Conekta flow, but using Mercado Pago as the payment processor. The frontend tokenizes card data using the Mercado Pago JS SDK and submits the resulting token along with card metadata to this endpoint. The server creates the MikroTik hotspot user first, then charges the card — rolling back the user if the charge fails. On success, optional Telegram notifications can be sent to the operator's configured chat.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/v1/payments/pagar-mercado-pago` | Process a Mercado Pago card payment and provision hotspot access |
| `GET` | `/api/v1/payments/estado-pago/{payment_id}` | Check the current status of a Mercado Pago payment by its ID |

## POST /api/v1/payments/pagar-mercado-pago

### Headers

<ParamField header="X-API-Key" type="string" required>
  Router API Key in the format `jwt_<token>`. Identifies the company (which must have a configured Mercado Pago access token) and the target MikroTik router.
</ParamField>

### Body

The request body accepts both canonical field names and their Spanish aliases. Both forms are valid.

<ParamField body="producto_id" type="integer" required>
  ID of the product to purchase. Obtain from `GET /api/v1/catalogo_perfiles_venta`. Also accepted as `product_id`.
</ParamField>

<ParamField body="token" type="string" required>
  Card token generated by the Mercado Pago JS SDK (`mp.createCardToken`). Single-use; expires shortly after creation.
</ParamField>

<ParamField body="payment_method_id" type="string" required>
  Mercado Pago payment method identifier returned by the SDK when the card is detected. Examples: `"visa"`, `"master"`, `"amex"`.
</ParamField>

<ParamField body="monto" type="float" required>
  Transaction amount in the product's currency. Also accepted as `transaction_amount`.

  <Warning>
    This value must exactly match the product's `precio` field (tolerance ±0.01). If the amounts differ, the server returns `HTTP 400` with a mismatch error before any MikroTik user is created or any charge is attempted. Always read the price from the catalog endpoint and pass it back unchanged.
  </Warning>
</ParamField>

<ParamField body="email_cliente" type="string" required>
  Customer's email address. Passed to Mercado Pago as the payer email. Must be a valid email format. Also accepted as `customer_email`.
</ParamField>

<ParamField body="nombre_cliente" type="string" required>
  Full name of the customer. Stored in the transaction record and included in Telegram notifications. Also accepted as `customer_name`.
</ParamField>

<ParamField body="issuer_id" type="string">
  Card issuer (bank) ID returned by the Mercado Pago SDK. Optional but recommended for improved approval rates.
</ParamField>

<ParamField body="cuotas" type="integer" default="1">
  Number of installments for the payment. Pass `1` for a single full charge. Also accepted as `installments`.
</ParamField>

<ParamField body="telefono_cliente" type="string">
  Customer phone number. Optional. Also accepted as `customer_phone`.
</ParamField>

<ParamField body="device_id" type="string">
  Mercado Pago device fingerprint ID, obtained from `mp.getIdentificationTypes()` or the `MP_DEVICE_SESSION_ID` cookie. Helps reduce fraud rejections.
</ParamField>

<ParamField body="payer" type="object">
  Additional payer information object to pass directly to the Mercado Pago API. If not provided, the server constructs a minimal payer with `{"email": customer_email}`. Useful for supplying identification type/number when required by the country's regulations.

  ```json
  {
    "email": "juan@example.com",
    "identification": {
      "type": "RFC",
      "number": "PEPJ901231..."
    }
  }
  ```
</ParamField>

<ParamField body="tipo_usuario" type="string" default="usuario_contrasena">
  Type of hotspot credentials to generate. Also accepted as `user_type`.

  - `"usuario_contrasena"` — (default) Username + password pair.
  - `"pin"` — Single 6-digit numeric PIN (empty password).
</ParamField>

<ParamField body="mac_cliente" type="string">
  MAC address of the client device. Required for auto-connection. Also accepted as `mac_address`.
</ParamField>

<ParamField body="ip_cliente" type="string">
  Current IP of the client device. Used during auto-connection. Also accepted as `ip_address`.
</ParamField>

<ParamField body="conexion_automatica" type="boolean" default="false">
  When `true`, attempts to auto-authenticate the device on MikroTik after a successful payment. Requires `mac_cliente`. Also accepted as `auto_connect`.
</ParamField>

### Response

On success the endpoint returns `HTTP 200`.

<ResponseField name="success" type="boolean">
  Always `true` on success.
</ResponseField>

<ResponseField name="id_transaccion" type="string">
  Mercado Pago payment ID (numeric, stored as string). Use this for `GET /api/v1/payments/estado-pago/{payment_id}`.
</ResponseField>

<ResponseField name="estado_pago" type="string">
  Mercado Pago payment status. `"approved"` for successful instant payments. May be `"pending"` in some cases (see `advertencia` field).
</ResponseField>

<ResponseField name="tipo_usuario" type="string">
  The user type used to generate credentials: `"usuario_contrasena"` or `"pin"`.
</ResponseField>

<ResponseField name="usuario_hotspot" type="object">
  WiFi credentials to display to the customer.

  <Expandable title="usuario_hotspot fields">
    <ResponseField name="usuario_hotspot.usuario" type="string">
      Hotspot username (alphanumeric for `usuario_contrasena`, 6-digit PIN for `pin` type).
    </ResponseField>

    <ResponseField name="usuario_hotspot.contrasena" type="string">
      Hotspot password. Empty string `""` for `pin` type.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="producto" type="object">
  Purchased product summary.

  <Expandable title="producto fields">
    <ResponseField name="producto.nombre" type="string">
      Product name.
    </ResponseField>

    <ResponseField name="producto.precio" type="float">
      Price that was charged.
    </ResponseField>

    <ResponseField name="producto.moneda" type="string">
      Currency code (e.g., `"MXN"`).
    </ResponseField>

    <ResponseField name="producto.perfil_mikrotik" type="string">
      MikroTik profile applied to the created hotspot user.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="cliente" type="object">
  Customer information from the request.

  <Expandable title="cliente fields">
    <ResponseField name="cliente.nombre" type="string">
      Customer name.
    </ResponseField>

    <ResponseField name="cliente.email" type="string">
      Customer email.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="mercado_pago" type="object">
  Mercado Pago-specific payment details.

  <Expandable title="mercado_pago fields">
    <ResponseField name="mercado_pago.payment_id" type="string | integer">
      Mercado Pago's unique payment identifier.
    </ResponseField>

    <ResponseField name="mercado_pago.status" type="string">
      Raw payment status from Mercado Pago (e.g., `"approved"`, `"pending"`).
    </ResponseField>

    <ResponseField name="mercado_pago.status_detail" type="string">
      Detailed status code from Mercado Pago (e.g., `"accredited"`, `"pending_contingency"`). Useful for debugging declined payments.
    </ResponseField>

    <ResponseField name="mercado_pago.installments" type="integer">
      Number of installments for this payment, as submitted.
    </ResponseField>

    <ResponseField name="mercado_pago.payment_method" type="object">
      Payment method object returned by Mercado Pago. Contains card type, brand, and last four digits.
    </ResponseField>

    <ResponseField name="mercado_pago.raw_response" type="object">
      Full raw response from Mercado Pago's payment API. Useful for debugging. Do not display to end users.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 UTC datetime of transaction completion.
</ResponseField>

<ResponseField name="auto_conexion" type="object">
  Auto-connection result. Same structure as the Conekta endpoint.

  <Expandable title="auto_conexion fields">
    <ResponseField name="auto_conexion.estado" type="string">
      `"conectado"` or `"no_conectado"`.
    </ResponseField>

    <ResponseField name="auto_conexion.mac" type="string">
      MAC address used in the connection attempt.
    </ResponseField>

    <ResponseField name="auto_conexion.ip" type="string">
      IP address used in the connection attempt.
    </ResponseField>

    <ResponseField name="auto_conexion.mensaje" type="string">
      Human-readable status message.
    </ResponseField>

    <ResponseField name="auto_conexion.verificado" type="boolean">
      Whether the connection was verified in MikroTik's active session table.
    </ResponseField>

    <ResponseField name="auto_conexion.session_id" type="string">
      MikroTik session ID. Only present when `verificado` is `true`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="advertencia" type="string">
  Only present when `estado_pago` is `"pending"`. Contains a human-readable warning message explaining that the payment is awaiting confirmation.
</ResponseField>

## GET /api/v1/payments/estado-pago/{payment_id}

Check the current status of a Mercado Pago payment. Useful for polling pending payments or verifying a completed transaction.

### Headers

<ParamField header="X-API-Key" type="string" required>
  Router API Key. The company associated with the key must have Mercado Pago configured.
</ParamField>

### Path Parameters

<ParamField path="payment_id" type="integer" required>
  The numeric Mercado Pago payment ID returned in `mercado_pago.payment_id` from the payment response.
</ParamField>

### Response

```json
{
  "success": true,
  "payment_id": 87654321,
  "status": "approved",
  "status_detail": "accredited",
  "amount": 15.0,
  "currency_id": "MXN",
  "date_approved": "2024-07-01T16:45:01.000Z",
  "date_last_updated": "2024-07-01T16:45:01.000Z"
}
```

## Error Codes

| Status | Trigger |
|--------|---------|
| `400 Bad Request` | Mercado Pago not configured for this company, or `transaction_amount` does not match the product price (±0.01 tolerance). |
| `402 Payment Required` | Payment was rejected by Mercado Pago (e.g., insufficient funds, declined by issuer). MikroTik user is rolled back. |
| `404 Not Found` | `producto_id` does not exist or does not belong to the authenticated company. |
| `500 Internal Server Error` | Failed to create MikroTik user, unexpected error from Mercado Pago, or internal server fault. |

<Note>
  When the company has Telegram notifications enabled (`notificaciones_telegram = true`), this endpoint sends a background notification to the configured `telegram_chat_id` on both successful payments and rejected ones (HTTP 402). These notifications are sent asynchronously and do not affect the API response time.
</Note>
